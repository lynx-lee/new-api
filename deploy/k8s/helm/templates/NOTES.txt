1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ .host }}{{ $.Values.ingress.paths[0].path }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "ai-bridge.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo "ai-bridge URL: http://$NODE_IP:$NODE_PORT"
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status by running:
           kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "ai-bridge.fullname" . }}
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "ai-bridge.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo "ai-bridge URL: http://$SERVICE_IP:{{ .Values.service.port }}"
{{- else if contains "ClusterIP" .Values.service.type }}
  kubectl port-forward \
    --namespace {{ .Release.Namespace }} \
    svc/{{ include "ai-bridge.fullname" . }} \
    {{ .Values.service.port }}:{{ .Values.service.port }}
  # Then access: http://localhost:{{ .Values.service.port }}
{{- end }}

2. Metrics endpoint (if enabled):
   kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "ai-bridge.fullname" . }} 9090:9090
   # Access: http://localhost:9090/metrics

3. For production deployment, see deploy/k8s/README.md for full guide.

4. Architecture overview:
   Ingress → Service → Deployment ({{ .Values.replicaCount | default 2 }} replicas)
     → External Redis Cluster / PostgreSQL Cluster / Prometheus

5. IMPORTANT for cluster mode:
   Ensure SESSION_SECRET is set consistently across all pods.
   Current secret: ai-bridge-{{ .Release.Name }}-secret
